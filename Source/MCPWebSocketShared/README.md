# MCPWebSocketShared

Wire protocol types for the WebSocket relay, shared between the iOS client adapter ([MCPWebSocket](../MCPWebSocket/README.md)) and the Lambda relay server ([MCPWebSocketRelay](../MCPWebSocketRelay/README.md)).

## Overview

This target defines the `Codable` message types exchanged over the WebSocket connection and stored in DynamoDB. It has **no dependencies**, so both iOS and server targets can import it without pulling in unnecessary frameworks.

## Types

### RelayMessage

Top-level envelope with three variants:

```swift
enum RelayMessage: Codable, Sendable {
    case request(RelayRequest)    // MCP client → iOS app
    case response(RelayResponse)  // iOS app → MCP client
    case control(RelayControlMessage) // Keep-alive, errors
}
```

### RelayRequest / RelayResponse

Carry MCP JSON-RPC bodies as raw `Data` (base64-encoded in JSON), avoiding any coupling to MCP request/response types:

```swift
struct RelayRequest: Codable, Sendable {
    let correlationId: String  // Matches request to response
    let body: Data             // Raw MCP JSON-RPC request
}

struct RelayResponse: Codable, Sendable {
    let correlationId: String  // Must match the original request
    let body: Data             // Raw MCP JSON-RPC response
}
```

### RelayControlMessage

```swift
enum RelayControlMessage: Codable, Sendable {
    case ping
    case pong
    case sessionEstablished(sessionId: String)
    case error(message: String)
}
```

## Design Decisions

- **Raw Data bodies**: MCP request/response types live in the `MCP` module. By carrying bodies as opaque `Data`, this module stays dependency-free and the relay doesn't need to parse MCP protocol details.
- **Base64 encoding**: `Data` fields are base64-encoded in JSON to ensure safe transport through JSON-based systems (WebSocket text frames, DynamoDB string attributes).
- **Correlation IDs**: UUIDs generated by the HTTP relay Lambda, used to match async request/response pairs through DynamoDB polling.
